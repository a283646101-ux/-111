# 中州养老项目学习指南

## 一、项目简介

这是一个**智慧养老管理系统**的后端项目，用于管理养老院的日常业务，包括床位管理、老人信息管理、房间管理、合同管理等功能。

---

## 二、技术栈（用到的技术）

| 技术 | 作用 | 简单理解 |
|------|------|----------|
| **Spring Boot 2.7.4** | 核心框架 | 帮我们快速搭建Web项目，不用写很多配置 |
| **MyBatis** | 数据库框架 | 帮我们操作数据库，把数据库表映射成Java对象 |
| **MySQL** | 数据库 | 存储数据的地方 |
| **Druid** | 数据库连接池 | 管理数据库连接，提高性能 |
| **Lombok** | 代码简化工具 | 自动生成getter/setter等方法，减少代码量 |
| **Swagger/Knife4j** | API文档工具 | 自动生成接口文档，方便测试接口 |
| **JWT** | 身份认证 | 用户登录后生成token，用于身份验证 |
| **阿里云OSS** | 文件存储 | 存储图片、文件等 |
| **Maven** | 项目管理工具 | 管理项目依赖、编译、打包 |

## 三、项目结构（三层架构）

```
zzyl/
├── zzyl-common/          # 公共模块（工具类、基础配置）
├── zzyl-service/         # 业务模块（核心业务逻辑）
├── zzyl-web/             # Web模块（控制器、启动类）
└── pom.xml               # 父工程配置
```

### 为什么这样设计？

这是经典的**三层架构**设计：

```
用户请求 → Controller（控制器）→ Service（业务层）→ Mapper（数据层）→ 数据库
                ↓
            返回结果给用户
```

**好处：**
- 各层职责清晰，便于维护
- 可以多人协作开发
- 代码复用性高

## 五、常见注解详解

### 5.1 @Autowired - 依赖注入

```java
@RestController
public class BedController {
    
    @Autowired
    private BedService bedService;  // 自动注入BedService
    
    // 相当于：Spring自动帮你创建了BedService对象并赋值
    // 不需要你手动 new BedService()
}
```

**理解：** `@Autowired` 就像是"自动装配"，Spring会自动把需要的对象给你准备好，你直接用就行。

**原理：**
1. Spring启动时，扫描所有带`@Service`、`@Component`等注解的类
2. 把这些类创建成对象（Bean），放入Spring容器
3. 当你用`@Autowired`时，Spring自动从容器中找到对应的对象注入

### 5.2 @RestController - 控制器

```java
@RestController
@RequestMapping("/bed")
public class BedController {
    // 这个类专门处理 /bed 开头的HTTP请求
}
```

**理解：** `@RestController` = `@Controller` + `@ResponseBody`
- 告诉Spring这是一个控制器
- 方法返回的数据会自动转成JSON格式

### 5.3 @RequestMapping - 路由映射

```java
@RestController
@RequestMapping("/bed")  // 类级别：所有方法都以/bed开头
public class BedController {
    
    @RequestMapping("/list")  // 完整路径：/bed/list
    public ResponseResult list() {
        return ResponseResult.success();
    }
}
```

### 5.4 @GetMapping / @PostMapping - HTTP方法

```java
@GetMapping("/list")    // 只响应GET请求，用于查询
@PostMapping("/add")    // 只响应POST请求，用于新增
@PutMapping("/update")  // 只响应PUT请求，用于修改
@DeleteMapping("/delete") // 只响应DELETE请求，用于删除
```

**理解：**
- **GET**：获取数据（查询）
- **POST**：提交数据（新增）
- **PUT**：更新数据（修改）
- **DELETE**：删除数据

### 5.5 @RequestBody - 接收JSON数据

```java
@PostMapping("/login")
public ResponseResult login(@RequestBody WechatLoginDto dto) {
    // @RequestBody 把前端传来的JSON自动转成Java对象
    // 前端传来：{"code": "xxx", "phone": "13800138000"}
    // 自动转成：dto.code = "xxx", dto.phone = "13800138000"
}
```

### 5.6 @Service - 服务层

```java
@Service  // 标记这是一个服务类，Spring会自动创建它的对象
public class BedServiceImpl implements BedService {
    // 业务逻辑写在这里
}
```

### 5.7 @Mapper - 数据访问层

```java
@Mapper  // 标记这是一个MyBatis的Mapper接口
public interface BedMapper {
    List<Bed> selectAll();  // 只需要定义方法，SQL写在XML文件中
}
```

### 5.8 @Data - Lombok注解

```java
@Data  // 自动生成 getter/setter/toString/equals/hashCode
public class BedVo {
    private Long id;
    private String bedNumber;
}
// 不用写几十行getter/setter代码了！
```

## 六、核心类详解

### 6.1 ResponseResult - 统一返回结果

```java
@Data
public class ResponseResult<T> {
    private int code;       // 状态码：200成功，其他失败
    private String msg;     // 提示信息
    private T data;         // 返回的数据（泛型，可以是任何类型）
    private Date operationTime;  // 操作时间
}

// 使用示例：
return ResponseResult.success();           // 成功，无数据
return ResponseResult.success(bedList);    // 成功，返回数据
return ResponseResult.error("操作失败");    // 失败
```

**前端收到的JSON格式：**
```json
{
    "code": 200,
    "msg": "操作成功",
    "data": {"id": 1, "bedNumber": "A001"},
    "operationTime": "2024-01-01 12:00:00"
}
```

### 6.2 DTO vs Entity vs VO

| 类型 | 全称 | 作用 | 使用场景 |
|------|------|------|----------|
| **Entity** | 实体类 | 对应数据库表 | 数据库操作 |
| **DTO** | Data Transfer Object | 接收前端传来的数据 | 接收请求参数 |
| **VO** | View Object | 返回给前端的数据 | 返回响应数据 |

**示例：**
```java
// Entity - 对应数据库表
public class Bed {
    private Long id;
    private String bedNumber;
    private Integer status;
    private Date createTime;
    private Date updateTime;
    // ... 数据库所有字段
}

// DTO - 接收前端参数（只包含需要的字段）
public class BedDto {
    private String bedNumber;  // 前端只传床位号
    private Long roomId;       // 和房间ID
}

// VO - 返回给前端（只返回需要的字段）
public class BedVo {
    private Long id;
    private String bedNumber;
    private String roomName;   // 关联查询的房间名称
    // 不返回createTime等内部字段
}
```

## 七、请求流程示例

以"查询床位列表"为例：

```
1. 前端发送请求
   GET /bed/list?roomId=1
   
2. Controller接收请求
   @GetMapping("/list")
   public ResponseResult list(@RequestParam Long roomId) {
       return bedService.getBedsByRoomId(roomId);
   }
   
3. Service处理业务
   public List<BedVo> getBedsByRoomId(Long roomId) {
       return bedMapper.selectByRoomId(roomId);
   }
   
4. Mapper查询数据库
   SELECT * FROM bed WHERE room_id = 1
   
5. 数据返回流程
   数据库 → Mapper → Service → Controller → 前端
```

## 十一、常见问题

### Q1: @Autowired注入失败？

**原因：** 没有加@Service注解，或者包扫描路径不对

**解决：**
```java
@Service  // 确保加了这个注解
public class BedServiceImpl implements BedService {
}
```

### Q2: 接口返回404？

**原因：** 路径写错了，或者没加@RestController

**检查：**
1. Controller类是否加了@RestController
2. @RequestMapping路径是否正确
3. 启动类是否扫描到了Controller包

### Q3: 数据库连接失败？

**检查：**
1. MySQL服务是否启动
2. application.yml中的数据库配置是否正确
3. 数据库是否存在

## 十二、学习建议

1. **先看懂三层架构**：Controller → Service → Mapper
2. **跟着一个功能走一遍**：比如"床位管理"，从Controller看到Mapper

4. **善用API文档**：访问 http://localhost:9995/doc.html 查看所有接口
5. **多动手**：尝试新增一个简单的接口，加深理解
